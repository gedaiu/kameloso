    Struct ThreadMessage

    Aggregate of thread message types.

    struct ThreadMessage ;

    This is a way to make concurrency message passing easier. You could use string literals to differentiate between messages and then have big switches inside the catching function, but with these you can actually have separate concurrency-receiving delegates for each.

    Inner structs

    NameDescription
    Abort Concurrency message askin the main thread to set the abort flag.
    BusMessage Concurrency message meant to be sent between plugins.
    Immediateline Concurrency message type asking to immediately send a message.
    PeekPlugins Concurrency message asking for a reference to the arrays of IRCPlugins in the current dialect.defs.IRCClient's plugin array.
    Pong Concurrency message type asking for a to-server dialect.defs.IRCEvent.Type.PONG event.
    Quietline Concurrency message type asking to quietly send a line to the server.
    Quit Concurrency message type asking to quit the server and the program.
    Reconnect Concurrency message asking to disconnect and reconnect to the server.
    Reload Concurrency message asking plugins to "reload".
    Save Concurrency message type asking to have plugins' configuration saved.
    Sendline Concurrency message type asking to verbosely send a line to the server.
    Teardown Concurrency message type asking for a plugin to shut down cleanly.
    WantLiveSummary Concurrency message asking the main thread to print a connection summary.
