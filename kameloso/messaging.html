    Module kameloso.messaging

    Functions used to send messages to the server.

    To send a server message some information is needed; like message type, message target, perhaps channel, content and such. dialect.defs.IRCEvent has all of this, so it lends itself to repurposing it to aggregate and carry them, through concurrency messages. These are caught by the concurrency message-reading parts of the main loop, which reversely parses them into strings and sends them on to the server.

    Example

    //IRCPluginState state;

chan(state, "#channel", "Hello world!");
query(state, "nickname", "foo bar");
mode(state, "#channel", "nickname", "+o");
topic(state, "#channel", "I thought what I'd do was, I'd pretend I was one of those deaf-mutes.");

    Having to supply the kameloso.plugins.common.IRCPluginState on every call can be avoided for plugins, by mixing in kameloso.plugins.common.MessagingProxy and placing the messaging function calls inside a with (plugin) block.

    Example

    IRCPluginState state;
auto plugin = new MyPlugin(state);  // has mixin MessagingProxy;

with (plugin)
{
    chan("#channel", "Foo bar baz");
    query("nickname", "hello");
    mode("#channel", string.init, "+b", "dudebro!*@*");
    mode(string.init, "nickname", "+i");
}

    Functions

    NameDescription
    askToLogImpl(state, line) Sends a concurrency message asking to print the supplied text to the local terminal, instead of doing it directly.
    chan(state, channelName, content, quiet) Sends a channel message.
    emote(state, emoteTarget, content, quiet) Sends an ACTION "emote" to the supplied target (nickname or channel).
    immediate(state, line) Immediately sends text to the server, verbatim. Skips all queues.
    invite(state, channel, nickname, quiet) Invites a user to a channel.
    join(state, channel, key, quiet) Joins a channel.
    kick(state, channel, nickname, reason, quiet) Kicks a user from a channel.
    mode(state, channel, modes, content, quiet) Sets a channel mode.
    part(state, channel, reason, quiet) Leaves a channel.
    privmsg(state, channel, nickname, content, quiet) Sends either a channel message or a private query message depending on the arguments passed to it.
    query(state, nickname, content, quiet) Sends a private query message to a user.
    quit(state, reason, quiet) Disconnects from the server, optionally with a quit reason.
    raw(state, line, quiet) Sends text to the server, verbatim.
    topic(state, channel, content, quiet) Sets the topic of a channel.
    whois(state, nickname, force, quiet) Queries the server for WHOIS information about a user.

    Aliases

    NameTypeDescription
    askToError Sends a concurrency message to the main thread to logger.error text to the local terminal.
    askToInfo Sends a concurrency message to the main thread to logger.info text to the local terminal.
    askToLog Sends a concurrency message to the main thread to logger.log text to the local terminal.
    askToTrace Sends a concurrency message to the main thread to logger.trace text to the local terminal.
    askToWarn Sends a concurrency message to the main thread to logger.warning text to the local terminal.
    askToWarning Simple alias to askToWarn, because both spellings are right.
    askToWriteln Sends a concurrency message to the main thread asking to print text to the local terminal.
    immediateline Merely an alias to immediate, because we use both terms at different places.
