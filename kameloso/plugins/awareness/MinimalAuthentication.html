    Template MinimalAuthentication

    Implements triggering of queued events in a plugin module.

    template MinimalAuthentication(bool debug_ = false, string module_ = __MODULE__) ;

    Most of the time a plugin doesn't require a full UserAwareness; only those that need looking up users outside of the current event do. The persistency service allows for plugins to just read the information from the dialect.defs.IRCUser embedded in the event directly, and that's often enough.

    General rule: if a plugin doesn't access IRCPluginState.users, it's probably going to be enough with only MinimalAuthentication.

    Contained Functions

    NameDescription
    onMinimalAuthenticationAccountInfoTargetMixin Replays any queued requests awaiting the result of a WHOIS. Before that, records the user's services account by saving it to the user's dialect.defs.IRCClient in the IRCPlugin's IRCPluginState.users associative array.
    onMinimalAuthenticationUnknownCommandWHOIS Clears all queued WHOIS requests if the server says it doesn't support WHOIS at all.

    NameDescription
    debug_ Whether or not to include debugging output.
    module_ String name of the mixing-in module; generally leave as-is.
