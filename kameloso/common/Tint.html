    Struct Tint

    Provides an easy way to access the *tint members of our KamelosoLogger instance logger.

    struct Tint ;

    Currently you need visibility of three things to be able to tint text;

    1. logger, as an instance of KamelosoLogger.
    2. KamelosoLogger itself, to cast logger to its subclass.
    3. settings, to know whether we want monochrome output or not.

    By placing this here where there is visibility of logger and settings, the caller need just import this.

    Methods

    NameDescription
    opDispatch (monochrome) Provides the string that corresponds to the tint of the std.experimental.logger.core.LogLevel that was passed in string form as the tint opDispatch template parameter.

    Example

    logger.logf("%s%s%s am a %1$s%4$s%3$s!", Tint.info, "I", Tint.log, "fish");

    If settings.monochrome is true, Tint.* will just return an empty string. The monochrome-ness can be overridden with Tint.*(false).

    Example

    import kameloso.logger : KamelosoLogger;

if (logger !is null)
{
    KamelosoLogger kl = cast(KamelosoLogger)logger;
    assert(kl);

    version(Colours)
    {
        assert(Tint.log is kl.logtint);
        assert(Tint.info is kl.infotint);
        assert(Tint.warning is kl.warningtint);
        assert(Tint.error is kl.errortint);
        assert(Tint.fatal is kl.fataltint);
    }
    else
    {
        assert(Tint.log == string.init);
        assert(Tint.info == string.init);
        assert(Tint.warning == string.init);
        assert(Tint.error == string.init);
        assert(Tint.fatal == string.init);
    }
}
